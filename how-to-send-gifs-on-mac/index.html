How to send GIFs on a Mac

Messages has GIF search built in. Everywhere else, you're mostly copy-pasting — unless you add something that fixes that.

How do I send a GIF in Messages on Mac?

Click the Apps button next to the message field — the icon just left of where you type — and choose #images. Pick a trending GIF, or click Find Images and search by keyword. Click a result to send it, or type a caption first and press Return.

  1. Open a conversation in Messages.
  2. Click the Apps button beside the compose field.
  3. Choose #images from the app drawer.
  4. Search by keyword, then click a GIF to send it — or add a caption first and press Return.

Received a GIF you want to keep? Control-click it in the conversation and choose Save to Downloads (or drag it straight to a folder). It stays animated as a file.

Sending GIFs everywhere else

#images only exists inside Messages. Open Slack, Mail, Notes, or a browser and you're back to copy-pasting from giphy.com or tenor.com — find one, copy it, paste it. That often works, but not always: some apps grab only the first frame of a copied GIF instead of the animated file, so what you paste is a still image with no warning that anything went wrong.

The reliable manual route is dragging the actual file: download the GIF first (on Giphy, right-click the image and save; on Tenor the download button gives you the real .gif), then drag it from Finder into the message or document. Dragging a file preserves the animation almost everywhere, because the receiving app gets a .gif file instead of whatever one frame the clipboard happened to carry. It's just a lot of steps for one reaction GIF.

Mojito's GIF search works in any app

Type :::excited anywhere macOS lets you type, and a Giphy-backed panel appears at your cursor. Arrow keys move through results, Return inserts. It pastes the GIF as an actual file rather than a clipboard image, so it animates properly in Slack, Discord, and anywhere else that accepts file drops — no static-frame surprises. GIF search is on by default and sends the text after ::: to Giphy to fetch results; turn it off in Settings → General if you'd rather keep everything local.

Fair questions

Why do GIFs paste as still images?

It depends on how the destination app reads the clipboard — some apps only grab the first frame from a copied GIF instead of the animated file. Dragging the actual file in, or using an inserter that pastes a file (like Mojito's ::: search), avoids the problem entirely.

Can I send GIFs in Slack without the /giphy command?

Yes — paste one from giphy.com or tenor.com directly, or type ::: plus a keyword in Mojito and it inserts as a file, which Slack renders animated. Slack's own /giphy command still works too; it's just one more way in.

Is there a GIF keyboard for Mac like the one on iPhone?

Not built in — the iPhone's GIF search lives in its keyboard, and macOS has no equivalent outside Messages. That gap is exactly what Mojito's ::: trigger fills: it behaves like a GIF keyboard that follows your cursor into any app.
