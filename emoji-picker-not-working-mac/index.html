Emoji picker not working on Mac? Here's the fix

One setting causes most of these reports. Here's how to check it, plus four other things that quietly block the picker.

Why won't the emoji picker open on my Mac?

Almost always it's one setting: System Settings → Keyboard → "Press 🌐 key to" isn't set to "Show Emoji & Symbols." Open that pane and check it first — this alone accounts for most "the Globe key does nothing" reports. If Control + Command + Space doesn't work either, something else is intercepting it. Keep reading.

Fix 1: Set the Globe key to open Emoji & Symbols

Go to System Settings → Keyboard. Find "Press 🌐 key to" and check the dropdown. If it's set to "Change Input Source," "Start Dictation," or "Do Nothing," that's the whole problem — switch it to "Show Emoji & Symbols" and the Globe key (or Fn key on older keyboards) will open the picker immediately. This setting resets more often than people expect, especially after a macOS update or a keyboard swap.

Fix 2: Use the shortcut, not the key

Control + Command + Space opens the emoji picker regardless of what the Globe key is set to — it's a separate, always-on shortcut. If that combination does nothing either, check System Settings → Keyboard → Keyboard Shortcuts for an app that has claimed the same combination. Some apps (screenshot tools, window managers, IDEs) bind over it without asking.

Fix 3: Rule out something blocking it system-wide

Two things are known to silently disable the emoji picker and other input helpers across the whole system: Terminal's "Secure Keyboard Entry" menu option, and certain VPN or endpoint-security clients — Cisco AnyConnect is a repeat offender. If either is active, quit or disable it and try the picker again. You'll know Secure Keyboard Entry is on because it's checked under Terminal → Secure Keyboard Entry in the menu bar.

Fix 4: Check for an off-screen panel

After connecting or disconnecting an external display, the emoji panel can remember a position that's now off-screen — it opens, you just can't see it. Disconnect the external monitor, or reset your layout in System Settings → Displays, and try again. Logging out and back in also resets the panel's remembered position.

Fix 5: The fallback that never breaks

Every app with a menu bar has Edit → Emoji & Symbols. It opens the same panel through a different path and isn't affected by any of the above — useful for telling whether the picker itself is broken or just the shortcut to it.

Skip the picker instead of fixing it

Mojito doesn't use the Globe key or the system panel at all — it autocompletes inline. Type :fire in any text field and a small picker appears right at your cursor; Return inserts 🔥. There's nothing for a macOS update to reset. Worth trying if you'd rather stop needing the fixes above than keep applying them.

If it's still acting up

Why did the Globe key work yesterday and not today?

Usually an input-source switch or a setting that reset after a macOS update — the Globe key's job (dictation, input source, or emoji) is one shared setting, and anything that changes your input sources can quietly reassign it. Check System Settings → Keyboard again; it's rarely a real bug.

Does the emoji picker work in every app?

Almost every app — except secure fields like password inputs, where macOS blocks all input helpers on purpose. If a picker, system or third-party, works in a password field, something's wrong.

How do I get the old ⌃⌘Space behavior back after an update?

Reassign it: System Settings → Keyboard → Keyboard Shortcuts, find whatever claimed the combination, and either disable that shortcut or change it. macOS won't restore ⌃⌘Space automatically once something else has taken it.
